# Proyecto Final: Algoritmo de búsqueda para la resolución del cubo Rubik

## 1. Introducción

El cubo Rubik es un rompecabezas tridimensional compuesto por diferentes piezas móviles que permiten generar millones de combinaciones posibles. Debido a la gran cantidad de estados que puede alcanzar, este problema es ampliamente utilizado en áreas como inteligencia artificial, teoría de grafos y estructuras de datos.

En este proyecto se implementó un resolvedor de cubo Rubik 3x3x3 utilizando el algoritmo de búsqueda BFS (Breadth First Search). El programa representa el cubo mediante estructuras híbridas de datos y permite realizar movimientos, visualizar el estado del cubo y encontrar una solución dentro de una profundidad limitada.

Además, el sistema incorpora visualización gráfica mediante Matplotlib para mostrar el estado del cubo y animar la solución paso a paso.

# 3. Explicación del programa

El programa fue desarrollado en Python utilizando programación orientada a objetos.

La clase principal `CuboRubikHibrido` representa las seis caras del cubo mediante matrices de 3x3 almacenadas dentro de un diccionario.

Cada movimiento del cubo se implementó mediante funciones independientes, permitiendo:

- Giros horarios.
- Giros antihorarios.
- Giros dobles.

El resolvedor utiliza BFS para explorar todos los estados posibles del cubo hasta encontrar una solución.

La búsqueda utiliza:

- Cola FIFO (`deque`).
- Conjunto de estados visitados.
- Copias profundas del cubo.

El sistema también incluye:

- Visualización gráfica del cubo.
- Animación paso a paso.
- Ingreso manual de estados.
- Resolución automática.

# 4. Funcionamiento del algoritmo BFS

El algoritmo Breadth First Search (BFS) explora primero todos los estados cercanos al estado inicial antes de avanzar a niveles más profundos.

## Funcionamiento general

1. Se agrega el estado inicial a una cola.
2. Se extrae el primer estado.
3. Se generan todos los movimientos posibles.
4. Se agregan nuevos estados no visitados.
5. El proceso continúa hasta encontrar la solución.

## Ventajas de BFS

- Garantiza encontrar la solución más corta.
- Es sencillo de implementar.
- Permite recorrer sistemáticamente el espacio de estados.

## Desventajas

- Consume mucha memoria.
- El tiempo de ejecución crece rápidamente.
- No es eficiente para profundidades muy grandes.

# 5. Tabla comparativa

| Característica | BFS | DFS |
|---|---|---|
| Tipo de recorrido | Por niveles | Por profundidad |
| Estructura utilizada | Cola (FIFO) | Pila (LIFO) |
| Garantiza solución mínima | Sí | No |
| Consumo de memoria | Alto | Bajo |
| Velocidad en problemas grandes | Más lenta | Más rápida en algunos casos |
| Complejidad temporal | O(V + E) | O(V + E) |
| Complejidad espacial | O(V) | O(h) |
| Adecuado para cubo Rubik | Sí, para soluciones cortas | Limitado |
| Riesgo de ciclos infinitos | Bajo usando visitados | Alto si no se controla |
| Facilidad de implementación | Alta | Alta |
